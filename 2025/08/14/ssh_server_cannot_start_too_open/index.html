SSH 服务无法启动：私钥权限过于开放

SSH 主机私钥权限过宽 导致 sshd 拒绝加载密钥，从而无法启动服务。

问题原因

  • /etc/ssh/ssh_host_rsa_key、ssh_host_ecdsa_key、ssh_host_ed25519_key 的权限是 0777（所有用户可读）。
  • sshd 要求 主机私钥文件必须仅 root 可读写（0600 权限）。

解决方法

  1. 修正权限

  2. 确保属主是 root

  3. 重启 SSH 服务

  4. 验证状态

额外建议

  • 如果密钥文件被错误权限暴露过（比如在系统运行时很久都是 0777），出于安全考虑，建议重新生成：
