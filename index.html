Insecure/STARTTLS/SSLTLS 测试邮箱搭建

نُشر في 2021-02-20
  1. 安装Hmailserver(https://www.hmailserver.com/), 需要提前安装.netformwork；
  2. 按照提示设置密码；
  3. 使用hMailServer Administrator连接到hmailserver, 密码是安装的时候设置的；
  4. 首先添加一个domain；
  5. 接着添加几个账号，设置好密码；
  6. 生成测试证书，可以去https://myssl.com/create_test_cert.html
  7. 可以看到以下三个文件：
    cert.pem对应服务器证书
    full_chain.pem对应CA证书
    private.key对应私钥
  8. 回到hMailServer Administrator, 打开Settings->Advanced->SSL certificates；
  9. 点Add 新加一个证书；
  10. 给证书起一个名字，Certificate file 对应cert.pem，Private key file对应private.key；
  11. 然后保存即可；
  12. 切换到Settings->Advanced->TCP ports;
  13. 这里可以按照需要进行设置相应的端口:
    一般对应如下:
    SMTP: 25(Insecure)/587(STARTTLS)/465(SSLTLS).
    POP3: 110
    IMAP:143
  14. 对于测试部分，这里只是用到了SMTP的发送功能，收件功能没有特殊要求。
  15. 选中0.0.0.0/587/SMTP, Connection security 选择 STARTTLS(Required), SSL Certificate选择步骤10里面的证书， 然后保存，这样就配置好了STARTTLS;
  16. 同样，选中0.0.0.0/465/SMTP(如果没有参考PORT 25来配置一个就好)，Connection security 选择SSL/TLS, SSL Certificate选择步骤10里面的证书然后保存，这样就配置好了SSL/TLS；
  17. 到此，邮件服务器就已经配置完成；
  18. 使用邮件客户端进行测试，可以用thunderbird 或者foxmail;
  19. 分别指定不同的SMTP发送服务器设置；
  20. 确保可以收到邮件；
  21. 检查hMailSever的log，可以看到对应的加密log:

解决Could not create SSL/TLS secure channel

نُشر في 2020-10-20

System.Net.WebException: The request was aborted: Could not create
SSL/TLS secure channel.

在我们的案例中，我们在使用软件供应商，因此我们无权修改.NET代码。除非有更改，否则.NET 4显然不会使用TLS v 1.2。

我们的解决方法是将SchUseStrongCrypto密钥添加到注册表中。您可以将以下代码复制/粘贴到扩展名为.reg的文本文件中并执行它。它充当了我们解决该问题的”补丁”

Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Microsoft.NETFramework\v4.0.30319]
“SchUseStrongCrypto”=dword:00000001

[HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft.NETFramework\v4.0.30319]
“SchUseStrongCrypto”=dword:00000001

GT-AC5300 Koolshare 官改安装Entware

نُشر في 2020-10-14
  1. 首先插入ex3/ex4 usb disk一枚。
  2. 创建对应的Entware文件夹:
    mkdir /mnt/sda1/entware
  3. 连接路由器的ssh，输入以下命令：
    cat << EOF > /tmp/script_usbmount.tmp
ln -nsf /mnt/sda1/entware /tmp/opt
EOF
nvram set script_usbmount="`cat /tmp/script_usbmount.tmp`"

cat << EOF > /tmp/script_usbumount.tmp
EOF
nvram set script_usbumount=”cat /tmp/script_usbumount.tmp“

nvram commit
4. 重启路由器
5. 重启之后查看文件夹/tmp/opt是否存在以及是否可以打开。
6. 在ssh里面执行下面的命令：

wget -O - http://bin.entware.net/aarch64-k3.10/installer/generic.sh | sh
  1. 完成后输入opkg update检查是否有有效输出。
    END
